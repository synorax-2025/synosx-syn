# SynOSX 导航体系说明

> 版本：v1.0
> 更新时间：2026-01-23
> 适用范围：SynOSX 官网（index / manifest / whitepaper / cases 等）

---

## 一、设计背景

SynOSX 并不是一个普通的官网，而是一套**以治理结构为核心的系统级产品**。因此，导航系统不能只是“能点”，而必须满足以下目标：

* 区分【叙事页面】与【结构页面】
* 适配桌面端与移动端完全不同的使用方式
* 避免 iOS 滚动陷阱（drawer / list 内滚动失败问题）
* 保留长期可演进的结构弹性

最终形成了 **Orb + Nav + sx.menu + sx.drawer** 的组合体系。

---

## 二、核心组件总览

| 组件            | 定位            | 是否全站使用 | 主要平台    |
| ------------- | ------------- | ------ | ------- |
| **Orb**       | 全局入口（意识入口）    | 是      | 仅移动端    |
| **sx.nav**    | 顶栏可见性与 CTA 宪法 | 是      | 桌面 + 移动 |
| **sx.menu**   | 横向 Tabs 控制台   | 否      | 结构型页面   |
| **sx.drawer** | 竖排叙事菜单        | 否      | 首页（移动端） |

---

## 三、Orb（全局入口）说明

### 1. Orb 是什么？

Orb 是 **SynOSX 的全局意识入口**，而不是菜单本身。

* 不承载导航结构
* 不负责展示内容
* 只负责：**在当前页面语境下，唤起合适的控制界面**

> Orb = 入口，不是菜单

### 2. Orb 显示规则（锁定）

| 设备            | 是否显示 Orb | 原因                    |
| ------------- | -------- | --------------------- |
| 桌面端（>= 981px） | ❌ 不显示    | 顶栏 + Hover + Tabs 已足够 |
| 移动端（<= 980px） | ✅ 显示     | 单手操作，无 Hover          |

### 3. Orb 的行为逻辑

Orb 本身**不决定打开什么**，而是由页面语境决定：

| 页面类型                  | Orb 点击后行为         |
| --------------------- | ----------------- |
| 首页（index）             | 打开 `sx.drawer v2` |
| Manifest / Whitepaper | 打开 `sx.menu`      |

---

## 四、sx.nav（顶栏宪法）说明

### 1. sx.nav 的职责

`sx.nav.css` **不负责任何菜单或抽屉视觉**，只负责：

* 顶栏内容的可见性规则
* 桌面 / 移动端的 CTA 策略
* 与 sx.menu / sx.orb 的协作边界

### 2. 顶栏可见性宪法

| 状态  | 行为                |
| --- | ----------------- |
| 桌面端 | 显示 `.nav-desktop` |
| 移动端 | 隐藏 `.nav-desktop` |

### 3. CTA 策略（延续旧设计）

| 设备  | CTA 行为                        |
| --- | ----------------------------- |
| 桌面端 | 主 CTA + 次 CTA                 |
| 移动端 | 仅主 CTA，次 CTA 进入 Menu / Drawer |

---

## 五、sx.menu（横向 Tabs 控制台）说明

### 1. sx.menu 的定位

`sx.menu` 是**结构型页面使用的横向控制台**：

* Manifest
* Whitepaper
* 未来的规范 / API / 文档类页面

它解决的是：

> “我在一个结构密集的页面中，如何快速定位内容？”

### 2. 设计特征

* 横向 Tabs（语义视角）
* 单一滚动容器（避免 iOS 假滚动）
* 浮层面板（非贴底）

### 3. 不适用页面

* 首页（叙事页面）
* 任何需要完整纵向阅读节奏的页面

---

## 六、sx.drawer（竖排叙事菜单）说明

### 1. 当前定位（v2 瘦身后）

`sx.drawer` **不再作为全站通用方案**，仅服务于：

* 首页（index）
* 移动端

### 2. 使用原因

* 首页是叙事型页面，不适合 Tabs
* 竖排结构更符合「浏览 / 感受」
* Drawer 可承载 Use Cases / Narratives 等内容块

### 3. 行为规则

* 从底部上浮（Bottom Drawer）
* 不贴底，保留边框与呼吸空间
* 展开 Use Cases 时允许高度变化，但不遮挡顶部核心内容

---

## 七、页面 × 组件 对照表（最终形态）

| 页面         | 桌面端            | 移动端             |
| ---------- | -------------- | --------------- |
| 首页         | 顶栏 + CTA       | Orb → sx.drawer |
| Manifest   | 顶栏 Tabs / Menu | Orb → sx.menu   |
| Whitepaper | 顶栏 Tabs / Menu | Orb → sx.menu   |
| Cases      | 顶栏导航           | Orb → 轻量 Menu   |

---

## 八、最终设计原则（锁定）

1. **入口与结构分离**（Orb ≠ Menu ≠ Drawer）
2. **叙事页面不用结构工具**
3. **结构页面不用叙事抽屉**
4. **iOS 滚动优先稳定性**
5. **允许不同页面拥有不同导航体验**

---

## 九、结语

这一套导航体系并非“组件堆叠”，而是：

> **围绕页面语义、设备特性与长期演进能力构建的系统级导航架构。**

它允许 SynOSX 在未来增加更多页面、更多治理模块，而不必推翻导航基础。

—— SynOSX UI / Interaction Architecture
