# 移动端顶栏（Mobile Topbar）— 锁死规则

### 一、 布局结构
* **左：** Logo
* **右：** ☰ Menu
* **右：** 1 个主 CTA（Primary CTA）

> **顶栏永远只允许 1 个 CTA。**

⚠️ **不存在「返回不算 CTA」这种灰色空间**
只要是按钮，就是 CTA，占名额。

---

### 二、 ✅ Drawer（Menu）里放什么
Menu = 路径 / 目录 / 次级动作集合。只放：

1. **所有 返回类**
   * 返回首页
   * 返回目录
   * 返回判例
2. **目录 / 内容入口**
   * Contents / Ch.1…Ch.N（可折叠）
   * Use Cases / Narratives
3. **非核心、低频行为**
   * Manifest / Whitepaper
   * Contact / About

---

### 三、 唯一判断标准
如果用户什么都不看，只点一个按钮就走——

* **你希望他点哪个？**
* 👉 **那个按钮 = 唯一允许出现在顶栏的 CTA**
* 其余所有按钮 **自动降级进 Menu**

---

### 四、 你原文中需要“修正”的地方（重点）
你之前有一句逻辑上会误导未来的你：
> 「可选 1 个 Secondary（返回）」 —— **❌ 这句话要删掉**

因为在手机端：
1. 用户只看到「按钮」。
2. 不会区分 Primary / Secondary 的语义。
3. 视觉上就是“两个 CTA 挤在右边”，这正是你现在看到「挤、换行、不好看」的根源。

---

### 五、 结合你现在的页面，给你唯一正确答案

#### 1. Index / Landing
* **顶栏主 CTA：** 预约技术对话
* **Menu：** Manifest, Whitepaper, Contact / About

#### 2. Manifest / Whitepaper
* **顶栏主 CTA：** 开始阅读
* **Menu：** 返回首页, Contents（Ch.1…Ch.N）, 切换 Manifest / Whitepaper

#### 3. Cases（判例列表）
* **顶栏（二选一）：** ✅ 无 CTA（最干净） 或 返回首页
* **Menu：** Narratives, Manifest, Whitepaper

#### 4. Narratives / Narrative-case（你最纠结的）
* **顶栏（只能选一个）：** ← 返回目录 或 ← 返回判例
* 👉 **选哪个，取决于你希望用户“下一步去哪”。**
* **Menu：** 另一个返回（没被选中的那个）, 返回首页, Manifest / Cases

---

### 六、 为什么「另一个返回必须进 Menu」（这是关键认知）
因为在手机上：
* **顶栏 = 行动入口**
* **Menu = 路径选择**

你现在看到的问题（右边按钮字太多、挤、换行、看起来不高级）不是 CSS 问题，也不是字体问题，而是：
👉 **你把“路径选择”塞进了“行动入口”**

---

### 七、 最终裁决（可直接写进你的设计原则文档）
**移动端顶栏 = Logo + Menu + 1 个主 CTA**

其余一切动作（包括所有返回）全部进入 Menu。
Menu 负责“去哪”
顶栏负责“现在做什么”
